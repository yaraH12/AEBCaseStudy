AEB Case Study — Model Representativity Evaluation for Automotive Simulation
This repository contains the case study developed to evaluate an automated model representativity assessment methodology applied in an automotive context.
The goal is to support engineers and researchers in selecting the most appropriate simulation model for a given need, while reducing trial‑and‑error and improving consistency across development phases.
The case study focuses on an AEB (Autonomous Emergency Braking) system, a key functionalitity of ADAS (Advanced Driver Assistance Systems). 
We want to select the most representative model of our simulation need accross 5 vehicle dynamics variants.

The system is modeled in AADL. Requirements are defined in ALISA ".reqspec" files. Each set of requirements refers to an AADL component using the "for" keyword.
Each requirement is linked to one of the RAS factors. To check them, we built verification plans (VPs) that contains all necessary claims. 
These VPs are then assured using the Assurance Plans declared in the Assurance Cases.


To test the Code:
In OSATE, with AADL perspective, in the "Assurance View Window", you will find all the defined assurance cases.
Right click on the one you want to check and click on "verify all" to obtain the claim list.
This list shows all requirements and their status (successful, fail, error).
